Browser Fingerprinting

Below lists all the information that the browser has with your current security settings. If information is missing or not accurate, that is a good thing! Some stats will vary based on extensions and privacy settings.

Displays

These properties contain info related to the display. They are used for features like responsiveness, auto-resize, and to dynamically show or hide content. They can also be used for identification. Using these properties, you can determine the exact position of the window across the viewer's monitors.

Property Value Description
sizeScreenW {{info.sizeScreenW}} The width of the screen in pixels
sizeScreenH {{info.sizeScreenH}} The height of the screen in pixels
sizeInW {{info.sizeInW}} The width of the browser window in pixels
sizeInH {{info.sizeInH}} The height of the browser window in pixels
sizeAvailW {{info.sizeAvailW}} The available width of the screen in pixels
sizeAvailH {{info.sizeAvailH}} The available height of the screen in pixels
scrColorDepth {{info.scrColorDepth}} The bit depth of the color palette for displaying images on the screen
scrPixelDepth {{info.scrPixelDepth}} The bit depth of the screen

Locked Properties

The above properties are available as soon as the user loads the page. They do not require any special permission. However, there are other properties that will require permission. These permissions can be used for a variety of unwanted purposes. They are listed below.

  • Microphone and Camera (Spying, everything implied)
  • Notifications (Ads)
  • Motion sensors (Determine your physical posture. Can be used to identify if you are paying attention to a website)
  • Protocol handlers (Hijack a protocol. For example, any email will now open in this website)
  • USB Devices / Serial Ports (Damage depending on what is plugged in)
  • Clipboard (Steal copied passwords and anything ready to be pasted)
  • Augmented Reality (Map your physical environment)
  • Window Management (Let it place browser windows on your display.)

While some of these are clearly bad, some do have practical uses. Its a good practice to deny these permissions unless you are sure a website needs it.
